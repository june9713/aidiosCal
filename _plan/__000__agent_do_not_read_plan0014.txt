# Plan 0014: 월간/주간 일정 시간 중복 조정 문제 해결

## 문제 현황
- 월간 일정과 주간 일정에서 스케줄 테이블의 일정 시간에 +9시간이 중복으로 적용됨
- 여러 번 수정 요청했으나 제대로 동작하지 않음
- 시간 처리 로직에서 일관성 부족

## 문제 원인 분석 필요 영역
1. **서버 측 (main.py)**:
   - `/gettimenow` 엔드포인트에서 9시간 추가 (256라인)
   - 스케줄 데이터베이스 저장/조회 시 시간 처리

2. **클라이언트 측 (main.js)**:
   - `koreaTimeOffset = 9 * 60 * 60 * 1000` 여러 위치에서 사용
   - `new Date(now.getTime() + koreaTimeOffset)` 중복 적용

3. **캘린더 로직 (calendar.js)**:
   - `getSchedulesForDate()` 함수에서 날짜 비교
   - `createWeeklyScheduleElement()` 함수에서 시간 계산

## 해결 계획
1. **현황 파악**: 콘솔 로그와 크롬 디버그 로그 추가
2. **원인 추적**: 시간 처리 과정에서 중복 적용 지점 식별
3. **수정 적용**: 일관된 시간 처리 로직 구현
4. **테스트**: 월간/주간 달력에서 정확한 시간 표시 확인

## 작업 순서
1. 디버깅 로그 추가
2. 시간 처리 흐름 추적
3. 중복 조정 제거
4. 일관성 있는 시간 처리 구현
